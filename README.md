# Mirvan LLC — mirvaninc.com

A static marketing site for Mirvan LLC, built for deployment on Cloudflare Pages.

## Stack

- Static HTML / CSS / vanilla JS — zero build step (Python only used to generate city pages from JSON and optimize images)
- Google Fonts: Fraunces (display), Inter Tight (sans), JetBrains Mono (mono)
- Black-and-white editorial aesthetic with light/dark mode
- All images self-hosted under /public/ for max reliability

## File structure

```
.
├── index.html                              # homepage
├── styles.css                              # all styles (shared by every page)
├── main.js                                 # theme toggle + scroll reveals
├── favicon.svg
├── robots.txt
├── sitemap.xml                             # auto-generated by build-cities.py
├── _headers                                # Cloudflare caching + security headers
├── cities.json                             # source data for city pages
├── build-cities.py                         # regenerates city pages from cities.json
├── optimize-images.py                      # generates hero + OG variants from raw photos
├── public/
│   ├── *_photo.jpg                         # raw source photos (high-res originals)
│   └── cities/                             # optimized output — DO NOT edit by hand
│       ├── {slug}-hero.jpg                 # 1920w, ~250-400KB (used on city page hero)
│       └── {slug}-og.jpg                   # 1200x630, ~100-150KB (used for link previews)
└── property-management-software/
    ├── index.html                          # hub page listing all cities
    └── {slug}/index.html                   # one page per city
```

## Adding or editing a city page

1. Edit `cities.json` (add an entry, change copy, etc.)
2. If swapping a photo: drop the new full-res JPG in `/public/`, update the filename mapping in `optimize-images.py`, then run `python3 optimize-images.py`
3. Run `python3 build-cities.py`
4. Commit and push — Cloudflare Pages deploys automatically

## Image pipeline

The site uses two image sizes per city:

- **Hero** (`/public/cities/{slug}-hero.jpg`) — full-bleed top-of-page image, 1920px wide
- **OG** (`/public/cities/{slug}-og.jpg`) — 1200×630 link-preview card for iMessage, Twitter, LinkedIn, etc.

To regenerate from source photos:

```bash
python3 optimize-images.py
```

This reads `/public/*.jpg`, generates optimized variants in `/public/cities/`, and prints byte savings.

## Features

- Mobile-first responsive layout
- Light/dark mode that follows system preference and can be overridden manually
- No flash of incorrect theme (theme set inline before paint)
- Accessible: skip link, focus styles, semantic HTML, reduced-motion support
- SEO-ready: meta tags, Open Graph + Twitter cards with proper dimensions, JSON-LD per city, sitemap.xml, robots.txt
- City pages with `LocalBusiness` schema, `areaServed`, and breadcrumb structured data
- Performant: no JS frameworks, fonts preconnected, images self-hosted, immutable cache headers via `_headers`

## Deploying to Cloudflare Pages

1. Push this directory to GitHub.
2. In Cloudflare → Workers & Pages → Create → Pages → Connect to Git.
3. Build settings:
   - **Framework preset:** None
   - **Build command:** *(leave blank — files are pre-generated)*
   - **Build output directory:** `/`
4. Save and Deploy.
5. Custom domains tab → add `mirvaninc.com` and `www.mirvaninc.com`.
